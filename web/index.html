cc-debrief

Post-session debrief for Claude Code transcripts. See where every token went — repeated reads, idle gaps, model overuse, hidden subagent cost — and get a personalised checklist of fixes for next session.

Pick your .claude folder, or a specific project folder under projects/

  • The full ~/.claude folderrecommended
    Walks every session JSONL plus your CLAUDE.md, all enabled skills, and settings.json. You'll get a sortable list of every session grouped by project, with full per-source attribution.
  • A specific project folder under ~/.claude/projects/
    Only that project's sessions appear in the list. CLAUDE.md and skill attribution stay empty because those configs live one level up.
Folder lives at %USERPROFILE%\.claude (Windows) or ~/.claude (macOS, Linux). Browsers can't auto-navigate to a specific path for security reasons; on Chrome/Edge, the picker remembers your last location so the second pick onwards is one click.

Multiple sessions — pick one

Click a project to load its most recent session. Click all to combine every session of that project into one cross-session report. Click ▸ to expand and pick a specific session.

    Privacy: nothing is uploaded. No LLM calls, no API key, no internet — your transcripts and the rendered report stay on this machine.
